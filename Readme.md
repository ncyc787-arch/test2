# PhoneMSG v0.5 — iMessage мессенджер с полным мостом

## Файлы
| Файл | Назначение |
|------|-----------|
| `index.js` | Точка входа, FAB, оболочка, события ST |
| `bridge.js` | Полный мост: инжект + скрытые маркеры + парсинг входящих |
| `engine.js` | Загрузка лорбука, генерация НПС, автосообщения |
| `state.js` | Глобальные настройки + per-chat состояние |
| `ui.js` | Список контактов, чат, настройки (через ⚙) |
| `api.js` | Extra LLM и Image API |
| `style.css` | Стили, floating/fullscreen, скрытие маркеров |
| `manifest.json` | Манифест |

## Как работает мост (v0.5)

### Канал 1: ИНЖЕКТ всей переписки (невидимо, в промпт бота)
После каждого сообщения — и твоего, и НПС — пересобирается `setExtensionPrompt` со всей историей:

```
[PHONEMSG — ПАРАЛЛЕЛЬНАЯ ПЕРЕПИСКА В ТЕЛЕФОНЕ]
ТвоёИмя параллельно с основным чатом переписывается в телефоне с НПС.
...
[📱 ТвоёИмя → Аякс в смс, 16:44]: где ты?
[📱 Аякс → ТвоёИмя в смс, 16:45]: дома, чё надо
[📱 ТвоёИмя → Аякс в смс, 16:45]: приходи
[END PHONEMSG]
```

Основной бот ST видит и твои, и ответы НПС. **В UI чата это НЕ ВИДНО.**

### Канал 2: МАРКЕРЫ отправки (скрыто через CSS)
При отправке смс в телефоне — в `context.chat[]` добавляется system-сообщение `[📱 Юзер отправил Аяксу в смс]: текст`. Оно помечено `extra.phonemsg_marker: 'outgoing'` и скрыто через CSS-класс `.phonemsg-hidden`.

Нужно чтобы бот получал real-time сигнал "юзер только что нажал send".

### Канал 3: ПАРСИНГ входящих тегов
Когда бот в основном чате пишет `[телефон:Аякс] ты где?`:
1. Текст сохраняется в переписку с Аяксом в телефоне
2. В основном чате тег заменяется на `*✉️ сообщение в телефоне: Аякс*`
3. На FAB загорается badge

### Канал 4: Автоконтакты
Бот пишет `[контакт:Саша:98765]` — Саша появляется в списке с пометкой 💬

## Порядок обновления инжекта (важно!)

После отправки твоего смс:
1. Сообщение юзера → в переписку
2. **rebuildConversationsInject()** — инжект обновлён с твоим сообщением
3. Маркер в c.chat[]
4. Генерация ответа НПС
5. Ответ НПС → в переписку  
6. **rebuildConversationsInject()** — инжект обновлён с ответом НПС

Вот это ключевое отличие от прошлой версии, где инжект обновлялся только из `handleSend` и ответы НПС туда не попадали.

## Настройки (⚙ в правом верхнем углу телефона)

- **Отображение**: floating / fullscreen (на мобиле всегда fullscreen)
- **Лорбук**: авто из чата/карточки ИЛИ по имени
- **Мост**: вкл/выкл, кастомные теги, кастомная заметка
- **API для НПС**: основной ST или свой Extra API
- **Image API**: свой или sillyimages
- **Автосообщения**: НПС пишут первыми

## Консоль (F12)

```js
phoneMsgOpen()        // открыть телефон
phoneMsgClose()       // закрыть
phoneMsgReload()      // перезагрузить контакты из лорбука
phoneMsgReprocess()   // пере-парсить всю историю чата
phoneMsgInject()      // принудительно пересобрать инжект
phoneMsgDebug()       // показать всё состояние
```

## Как проверить что работает

1. Открой F12 → Console. Должно быть:
   ```
   [PhoneMSG] FAB создан
   [PhoneMSG] Ready ✓ (N контактов)
   ```

2. Отправь смс в телефоне. В консоли:
   ```
   [PhoneMSG-Bridge] инжект обновлён: X строк
   [PhoneMSG-Bridge] маркер отправки → Аякс: текст
   [PhoneMSG-Bridge] инжект обновлён: X+1 строк  (после ответа НПС)
   ```

3. Запусти `phoneMsgDebug()` — должен показать `state.conversations` со всеми смс.

4. Если хочешь увидеть что уходит в промпт: посмотри `setExtensionPrompt` через `SillyTavern.getContext().extensionPrompts`.

## Как объяснить боту формат тегов

Добавь в карточку персонажа или в лорбук:

> Когда кто-то хочет написать {{user}} в мессенджер — используй формат `[телефон:Имя] текст`. Для добавления нового контакта используй `[контакт:Имя:ID]`.

## Установка

Положи все файлы в папку `public/scripts/extensions/third-party/PhoneMSG/`. В SillyTavern → Extensions → должно появиться `PhoneMSG`.
